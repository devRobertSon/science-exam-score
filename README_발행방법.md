# 과학 약점진단 리포트 — GitHub Pages 발행 방법

## 구성
- `index.html` : 학생이 개인 코드를 입력하는 화면. 코드로 SHA‑256 토큰을 계산해 `reports/{토큰}.pdf`로 이동한다.
- `reports/{토큰}.pdf` : 학생별 총괄 리포트. **PDF 자체가 학생 코드로 AES‑256 암호화**되어 있어, 코드를 모르면 내려받아도 열 수 없다.
- `404.html`, `robots.txt(noindex)` : 검색 노출 방지 + 잘못된 접근 처리.
- `_코드맵.csv`, `_salt.txt` : **비공개.** `.gitignore`로 깃 저장소에서 제외된다. 절대 커밋하지 말 것.

## 보호 구조 (2중)
1. **입력 게이트** : index.html에서 올바른 코드라야 자기 PDF 주소가 만들어진다(파일명은 추측 불가한 난수 토큰).
2. **PDF 암호** : 해당 PDF도 같은 코드로 암호화되어 있어, 브라우저가 열 때 코드를 한 번 더 묻는다.
   → 링크가 새어도, 파일을 직접 받아도, 코드 없이는 열람 불가.

## 발행 절차 (직접 실행)
1. GitHub에서 새 저장소 생성 (예: `science-reports`). 공개/비공개 모두 Pages 사용 가능하나 **URL은 공개**임에 유의.
2. 이 `발행` 폴더에서:
   ```
   cd "발행"
   git init
   git add .                # .gitignore 덕분에 _코드맵.csv·_salt.txt는 빠짐
   git commit -m "publish reports"
   git branch -M main
   git remote add origin https://github.com/<사용자>/<저장소>.git
   git push -u origin main
   ```
3. GitHub 저장소 → Settings → Pages → Source를 `main` 브랜치 `/ (root)`로 지정 → 저장.
4. 1~2분 뒤 `https://<사용자>.github.io/<저장소>/` 가 열린다. 이 주소를 학생에게 공유.
5. 커밋 전 `git status`로 `_코드맵.csv`·`_salt.txt`가 목록에 없는지 반드시 확인.

## 코드 배포
- `_코드맵.csv`에 학생별 코드가 있다. 학생에게 **사이트 주소 + 개인 코드**를 개별 전달한다(메신저·문자 등 1:1).
- 학생 사용법: 사이트 접속 → 코드 입력 → PDF 열릴 때 같은 코드 한 번 더 입력.

## 주의 (개인정보)
- GitHub Pages는 공개 웹이다. 암호화로 내용은 막지만 **링크+코드를 둘 다 받은 사람은 누구나 열람** 가능하다.
- 코드 재공유는 막을 수 없다. 진짜 접근통제(로그인·열람기록·만료)가 필요하면 인증되는 비공개 호스팅을 쓴다.
- 학생 이름은 파일명·URL에 넣지 않았다(토큰만 노출). 발행 전 **학부모 동의**를 받는 것을 권장한다.

## 코드 분실/교체
- 코드를 바꾸려면 해당 학생의 PDF를 새 코드로 다시 암호화하고(파일명 토큰도 새로 계산) 다시 커밋한다. 생성 스크립트는 담당자(Claude)에게 요청하면 재실행 가능.
